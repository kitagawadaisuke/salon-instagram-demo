## 1. 受け取り方の基本方針

- **パスワード共有より「管理者として招待」を優先**（LINE / Metaビジネスポートフォリオは招待で権限委譲できる）。
- **2段階認証（2FA）の受け取り先を必ず確認**。ログイン時の認証コードが担当のスマホ/メールに飛ぶと、こちらがログインできず詰まる。
- パスワードを受け取る場合は、チャット平文ではなく安全な方法（パスワードマネージャ共有・口頭等）で。

## 3. Meta / Facebook から受け取るもの

今回は **Instagram Login 方式**のため、IG側にFacebookページをつなぐことはしないでOK。ただしMeta Appを作るためにビジネスポートフォリオまわりの確認が要る。

- [ ] **既存の Meta ビジネスポートフォリオはあるか？**（あれば名称 / ID）
  - ある → こちらを **管理者として招待**してもらう（business.facebook.com → ビジネス設定 → ユーザー → 招待）
  - ない → こちらで新規作成するので作業不要（その旨だけ教えてもらう）
- [ ] **ai.labo0601 はどのFacebook/ポートフォリオに紐付いているか**（所有関係の確認）
- [ ] Facebookアカウント（id: 61590636153376）の位置づけ（IG垢の所有者か、ポートフォリオの管理者か等）

> Meta Developer 登録自体は自分のFacebookアカウントでも可能。担当のFBログインが必須とは限らない。鍵は「IG垢とポートフォリオの所有関係」なので、そこを確認する。

## 4. LINE から受け取るもの

LINE公式アカウントは既存（AI伴走LABO）。これを Messaging API 化して使う。**LINEは招待で権限委譲できる**ので、ログイン情報の共有は不要。

- [ ] **LINE公式アカウントの管理者として、こちらのLINE Business ID（メール）を招待**してもらう
  - LINE Official Account Manager → 設定 → 権限管理 → メンバー追加
- [ ] **LINE Developers コンソール**にも同様にアクセスできるよう招待 or 共有
  - （Messaging APIチャネルの作成・トークン発行はこちらで行う）
- [ ] LINE公式アカウントの **料金プラン**（フリー/ライト/スタンダード）
- [ ] こちらに付与する権限ロール（管理者 推奨）

> ※ チャネルアクセストークン・チャネルシークレットは、招待さえもらえれば **こちらで発行できる**ので、担当に発行させる必要はない。
